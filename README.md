# Data Modelling & UML Projects

This repository showcases my work in data modelling and system design using UML diagrams. These projects focus on applying theoretical concepts to real-world scenarios, with an emphasis on understanding system interactions, actors, and relationships.

## 📌 Tools Used
- Visual Paradigm
- UML (Use Case Diagrams)


## 🏧 ATM (Cash Dispenser) System

This project models the functionality of an ATM system using a Use Case Diagram.

### Key Features:
- Card holders can withdraw money after authentication
- Bank customers can:
- Check account balance
- Deposit money (authentication required)
- Maintenance operator:
  - Refills the ATM
  - Collects deposited cash

### What I Focused On:
- Defining the system boundary
- Identifying actors and their roles
- Mapping interactions between users and the system

## 🎓 University Registration System

This project represents a more complex system involving multiple actors and interactions.

### Key Features:
- Students:
  - Register for classes
  - Integrated billing system checks
- Professors:
  - Select courses to teach
  - Request course rosters
- Registrar/Admin:
  - Manages students, professors, and courses
  - Maintains the course catalogue

### What I Focused On:
- Modelling relationships between multiple actors
- Representing real-world processes in a structured way
- Understanding how different roles interact within a system

## 📈 Key Learning Outcomes
